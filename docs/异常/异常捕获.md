> 异常捕获介绍

React 16 使用了更具弹性的错误处理策略：错误边界。默认情况下，如果在组件的渲染或生命周期方法中抛出错误，整个组件树将从根目录卸载。这可以防止显示损坏的数据。但是，这可能不是理想的用户体验。
